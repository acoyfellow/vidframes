vidframes

Cost-aware video frame extraction, vision analysis, and audio transcription via Workers AI.

Quick start

bun install

# probe a video — see duration, resolution, frame estimates
bun run src/cli.ts probe video.mp4

# extract frames (scene detection by default — cheapest)
bun run src/cli.ts extract video.mp4 --output ./frames

# dry-run analysis — see how many frames + which model before spending
bun run src/cli.ts analyze video.mp4 --dry-run

# full run: extract + analyze + transcribe
CLOUDFLARE_ACCOUNT_ID=xxx CLOUDFLARE_API_TOKEN=xxx \
  bun run src/cli.ts run video.mp4 --prompt "Describe what's happening"

How it works

1. Extract

ffmpeg pulls frames from the video. Three modes:

  • scene — only on scene changes (default, fewest frames)
  • interval — every N seconds
  • keyframe — I-frames only

Frames are resized to a max dimension (default 512px) before analysis — smaller images mean fewer tokens.

2. Analyze

Each frame is sent to a Workers AI vision model. Concurrency-limited (default 3 parallel). Hard --max-frames cap prevents runaway costs.

Use --dry-run to see exactly how many frames will be analyzed before spending a cent.

3. Transcribe

Audio is extracted and split into 30s chunks. Each chunk is sent to Workers AI Whisper. Segments are reassembled with timestamps.

API

All logic lives in the library. The CLI is a thin wrapper.

FunctionDescription
probeVideo(path)Get duration, resolution, fps, codec
extractFrames(path, opts)Extract frames (scene/interval/keyframe, resize, limit)
estimateFrameCount(info, opts)Estimate frame count before extraction
analyzeFrame(path, opts)Send one frame to a vision model
analyzeFrames(frames, opts)Batch analyze with concurrency + maxFrames
analyzeVideo(path, opts)Extract + analyze + transcribe in one call
estimateAnalysis(info, extractOpts, analyzeOpts)Dry-run cost estimate
transcribeVideo(path, opts)Extract audio, chunk, transcribe via Whisper

Deploy

# build the site + deploy to vidframes.coey.dev
bun run deploy

The Worker serves the static site from site/dist. Custom domain is configured in wrangler.jsonc.
