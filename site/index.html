Quick start

Three commands to start

Probe a video for free. Dry-run to see costs. Run the full pipeline when ready.

bash — probe
bun run src/cli.ts probe video.mp4

# Video info:
#   duration: 15.0s
#   resolution: 1280x720
#   fps: 30.0
#   scene: ~3 frames
#   interval: ~3 frames
#   keyframe: ~30 frames
bash — dry-run
bun run src/cli.ts analyze video.mp4 --dry-run

# Dry run — cost estimate:
#   frames: 3
#   model: @cf/meta/llama-3.2-11b-vision-instruct
#   resize: 512px max dimension
#   transcription segments: ~1
#   Run without --dry-run to execute.
bash — full smart analysis
CLOUDFLARE_ACCOUNT_ID=xxx CLOUDFLARE_API_TOKEN=xxx \
  bun run src/cli.ts smart video.mp4 \
  --max-timestamps 3 \
  --prompt "Describe what's happening"
How it works

Four modes, one pipeline

All logic lives in the library. The CLI is a thin wrapper.

1. Extract

ffmpeg pulls frames from the video. Three modes:

  • scene — only on scene changes (default, fewest frames)
  • interval — every N seconds
  • keyframe — I-frames only

Frames are resized to a max dimension (default 512px) before analysis — smaller images mean fewer tokens.

2. Analyze

Each frame is sent to a Workers AI vision model. Concurrency-limited (default 3 parallel). Hard --max-frames cap prevents runaway costs.

Use --dry-run to see exactly how many frames will be analyzed before spending a cent.

3. Transcribe

Audio is extracted and split into 30s chunks. Each chunk is sent to Workers AI Whisper. Segments are reassembled with timestamps.

4. Smart

smartAnalyze flips the flow: transcribe first (cheap), then ask a text LLM "which timestamps have visual value?" (diagrams, slides, code). Extract 3-5 frames at those exact moments. Analyze only those.

10x cheaper than blind extraction.

Library API

All logic in userland

The CLI is a thin wrapper. Import and call directly.

typescript — smart analysis
import { smartAnalyze } from 'vidframes';

const result = await smartAnalyze('video.mp4', {
  maxTimestamps: 5,
  analyze: { prompt: 'Describe the diagram shown' },
  onProgress: (phase, detail) => console.log(`${phase}: ${detail}`),
});

// 1. transcribes audio (cheap whisper)
// 2. asks text LLM which timestamps have visual value
// 3. extracts 3-5 frames at those exact timestamps
// 4. analyzes only those frames with vision model
console.log(result.selectedTimestamps);
// [{timestamp: 120, reason: "whiteboard diagram"}, ...]
